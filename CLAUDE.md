# InstaSquare（インスタ用画像補正）

## プロジェクト概要

### 目的・ゴール
Instagram投稿用に、複数枚の写真を素早く加工できるWebアプリ。
1:1（フィード）と9:16（ストーリーズ／リール）の両方に対応し、位置調整・ズーム・回転・フィルターを指1本で完結させる。

### ユースケース
- スマホ（Chrome / Safari）で開いてその場で加工する
- 最大20枚を一度にアップ → 個別に位置/ズームを調整 → 一括フィルター → ZIPで一括DL

### 技術スタック
- 純粋なHTML / CSS / JavaScript（フレームワーク不使用）
- ライブラリ：JSZip（CDN）
- ブラウザAPI：FaceDetector API（Chrome系のみ・フォールバックあり）、createImageBitmap（EXIF回転）、Canvas 2D、Pointer Events
- 配信：GitHub Pages

### やらないこと・制約
- サーバー処理なし（すべてクライアント完結 = プライバシー安心）
- ログイン／保存機能なし（ブラウザを閉じたら状態消える）
- 動画は非対応（JPG / PNG / WebP のみ）
- 7枚以上の並び替え機能は今回見送り（必要になったら追加）
- ウォーターマーク機能は見送り

### デザイン方向性
- ダークテーマ（背景 #0a0a0f）＋ Instagramブランドカラーのグラデーション
- スマホファースト：sticky上部タブ、最小タップ44px、safe-area対応
- 余計な装飾を排し、操作優先

## 実装機能まとめ

### 必須機能
- アスペクト比 1:1 / 9:16 タブ切替（sticky top）
- ドラッグで位置調整（rotation適用後の画像基準）
- ズーム 1.0〜4.0x（ホイール / スライダー / ピンチ）
- 最大20枚アップロード

### 便利機能（実装済み）
| 番号 | 機能 | 実装方式 |
|---|---|---|
| ① | 三分割グリッド | ドラッグ中に表示（CSS） |
| ② | 90°回転 / 左右反転 | rotation 0/90/180/270, flipH |
| ③ | 彩度・コントラスト | ctx.filter ベース |
| ④ | EXIF自動回転 | createImageBitmap({imageOrientation:'from-image'}) |
| ⑤ | プリセット6種 | なし/映える/シック/フィルム/モノクロ/暖色/寒色 |
| ⑥ | ファイル名連番 | プレフィックス + 連番 + アスペクトタグ |
| ⑧ | Before/After切替 | beforeMode フラグでcanvas表示切替 |

### 既存機能（保持）
- 人物自動検出（FaceDetector API）
- saliency-basedフォールバック（肌色＋彩度＋輝度＋中心バイアス）
- 自動配置ボタンで再実行可能

## 重要な設計判断（次回の修正で迷ったとき用）

### 1. クロップ中心の座標系
**「rotation適用後の表示画像基準の 0..1 正規化値」** で `cropCx, cropCy` を保持。
- 回転すると軸の意味が変わるため、**rotation変更時は cropCx, cropCy を 0.5（中央）にリセット**
- flip時は `cropCx = 1 - cropCx`（左右反転後の同じ位置を維持）

### 2. zoom の定義
`zoom = 1.0` = 「画像が出力枠を完全に覆う最小サイズ」と定義。
- `baseScale = Math.max(outW/dispImgW, outH/dispImgH)`
- 実際のスケール = `baseScale * zoom`
- 最低 1.0、最大 4.0

### 3. ぼかし処理
- **プレビュー**：CSS filter blur（軽量・即時）
- **出力時**：ダウンスケール→アップスケールの反復方式（端フェードなし・artifact-free）
- 異なる方式だがプレビューは「概観確認」なので許容

### 4. 色フィルター（明るさ/コントラスト/彩度/プリセット）
- 出力時は ctx.filter で別キャンバスに描画してから合成（背景にblurがかからないように分離）

### 5. アスペクト切替時の挙動
- 全画像の cropCx, cropCy を 0.5、zoom を 1.0 にリセット
- 比率が変わるためクロップ位置は流用できない

### 6. EXIF対応
- `createImageBitmap(file, { imageOrientation: 'from-image' })` で取得
- 失敗時は `<img>` フォールバック（モダンブラウザは img も自動回転）

### 7. スマホタッチ操作
- Pointer Events で mouse / touch を統一
- 1本指 = ドラッグ、2本指 = ピンチ
- ピンチ→1本指残り時に新たな drag start として再アンカリング

## 既知のトレードオフ・注意点

- **FaceDetector API はChrome系のみ**：Safari/Firefoxではsaliencyフォールバックに自動切替（精度は落ちる）
- **プレビューと出力のぼかしが微妙に違う**：プレビューはCSS、出力はダウンスケール方式
- **回転時にクロップ中心がリセット**：意図的（軸が変わるため）。違和感あれば修正候補
- **20枚×フルサイズ処理は重め**：iPhone 13以降なら問題ないが古い端末は遅い可能性
