# InstaSquare（インスタ用画像補正）

## プロジェクト概要

### 目的・ゴール
Instagram投稿のためのスマホ向け画像加工Webアプリ。
1枚ずつ「ぼかし」と「明るさ」を調整し、1:1または9:16で個別ダウンロードする。

### ユースケース
- スマホ（Chrome / Safari）で開いてその場で加工
- 最大20枚を一度にアップ → 1枚ずつ位置調整・補正 → 個別DL

### 技術スタック
- 純粋なHTML / CSS / JavaScript（フレームワーク・ライブラリなし）
- ブラウザAPI：FaceDetector API（Chrome系、フォールバックあり）、createImageBitmap（EXIF回転）、Canvas 2D、Pointer Events、visualViewport
- 配信：GitHub Pages（sasukewebjob-ai/instasquare）

### やらないこと・制約
- サーバー処理なし、ログイン・保存なし（クライアント完結）
- 動画は非対応（JPG / PNG / WebP のみ）
- 過去に実装したが削除した機能（再追加禁止ではないが、機能追加は慎重に）：
  - 回転 / 反転 / Before-After / コントラスト / 彩度 / プリセット / ZIP一括 / ファイル名連番 / 自動配置ボタン / ズームスライダー（→ピンチのみ）

### デザイン方向性
- ダークテーマ + Instagramブランドカラーのグラデーション
- スマホファースト：sticky上部タブ、最小タップ44px、safe-area対応
- **スクロールなしで1画面に全要素が収まる**（実機テスト後に圧縮済み）

## 現在の機能

### アスペクト比
- 1:1（1080×1080）/ 3:4 縦長（1080×1440）/ 9:16（1080×1920）タブ切替（sticky top）
- タブは「比率＋用途」の2段表示（3タブでも390px幅に収まる）

### 編集（画像個別に保存）
- ドラッグで位置調整（指1本）
- ピンチで 1.0〜4.0x ズーム（指2本、マウスホイール対応）
- ぼかしスライダー（0〜20）
- 明るさスライダー（-100〜+100）

### 画像管理
- 最大20枚アップロード（クリック / ドラッグ&ドロップ）
- サムネ横列で切替・追加
- リセット（その画像のぼかし・明るさ・位置・zoomを初期化）
- この画像を削除 / 全クリア

### 出力
- 個別ダウンロード（JPG、ファイル名: `元名_1x1.jpg` / `元名_3x4.jpg` / `元名_9x16.jpg`）

## 重要な設計判断

### 1. 状態モデル：画像個別フィルター
ぼかし・明るさ・ズーム・クロップ位置はすべて **画像オブジェクト個別** に保持。
グローバルフィルターは持たない。
画像切替時、スライダーUIをその画像の保存値に同期。

```js
imageObj = {
  id, name, bitmap, thumbDataUrl,
  cropCx, cropCy,                    // ドラッグで動く 0..1
  autoSuggested: { cx, cy },          // リセット時の戻し先
  blur: 0,                            // 0〜20 個別
  brightness: 0,                      // -100〜100 個別
  zoom: 1.0                           // 1.0〜4.0 個別
};
```

### 2. プレビューサイズ：viewport動的計算
固定値ではなく、`viewport高さ − 他UIの実測高さ` で残り全部をプレビューに使う。
`visualViewport.resize` でiOSアドレスバーの出し入れにも追従。

### 3. zoom の定義
`zoom = 1.0` = 「画像が出力枠を完全に覆う最小サイズ」と定義。
`baseScale = Math.max(outW/bw, outH/bh)`、実スケール = `baseScale × zoom`。

### 4. クロップ位置の自動クランプ
`clampCrop(value, image, isX)` で画像が枠を覆う範囲内に強制。
zoom変更時・アスペクト切替時にも適用。

### 5. ぼかし処理
- **プレビュー**：CSS filter blur（軽量・即時）
- **出力時**：`ctx.filter blur(radius)`。radius = スライダー値 × (出力幅 / プレビュー表示幅) でプレビューと一致させる。ぼけ層をシャープ層の上に重ねて端の黒フチを防ぐ
- **`ctx.filter` 非対応環境（古いSafari等）**：目標半径から縮小率を逆算（`1 / max(1, radius/6)`、実測でChrome経路に近い強さ）し、半減の繰り返しで縮小→段階的拡大で戻すフォールバック。radius/6 の係数はピクセル検証（エッジ勾配比較）でチューニング済み

### 6. 明るさ処理
- ctx.filter brightness を別キャンバスに適用してから合成
- 0以外のとき**だけ**かける（無加工時の処理コスト回避）
- **`ctx.filter` 非対応環境**：`getImageData` でRGB各値を `1 + brightness/200` 倍するピクセル直接乗算フォールバック（CSS brightness() と同じ線形乗算）

### 7. アスペクト切替時の挙動
- 全画像の cropCx, cropCy = 0.5、zoom = 1.0 にリセット
- 軸の意味が変わるため位置情報を流用できない

### 8. EXIF対応
- `createImageBitmap(file, { imageOrientation: 'from-image' })` で取得
- 失敗時は `<img>` フォールバック

### 9. タッチ操作
- Pointer Events で mouse/touch を統一
- 1本指 = ドラッグ、2本指 = ピンチ
- ピンチ→1本指残り時に座標を再アンカリングしてドラッグ移行

### 10. UI圧縮（スクロールなし1画面）
- ヘッダー横並び・タグライン非表示
- フッターモバイル非表示
- ボタン4個を 2×2 グリッド（DLとクリアは全幅、リセット/削除は2列）
- container gap・padding を最小化

## 既知のトレードオフ

- **FaceDetector はChrome系のみ**：Safari/Firefoxではsaliencyフォールバック
- **古いSafari（iOS 17以前）のぼかしは近似**：縮小→拡大方式のため ctx.filter 経路と強さが完全一致はしない（実測で近い係数に調整済み）
- **20枚×フルサイズ処理は重め**：古い端末で遅い可能性
